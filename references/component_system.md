# Component System

## 目标

先锁组件系统，再批量出图。

## 必锁项

1. 页面原型
2. 组件族
3. 颜色 token
4. 字体 token
5. spacing token
6. 图表模板规则
7. 几何规则
8. 主区块占比
9. 对齐轴与连接关系

## 页面原型建议

1. 封面页
2. 诊断页
3. 样例/证据页
4. 双栏对比页
5. 阶段/流程页
6. 结构说明页
7. 场景页
8. 成交收口页

## 组件锁定规则

至少锁定：

- 面板风格
- 标题条样式
- 数字卡样式
- 标签样式
- 图表容器样式
- 连接线样式
- 关系节点样式
- 泳道/时间轴样式
- 表格容器样式

## 视觉特征锁定

如果使用代码生成，必须把关键视觉特征固化为 token 或结构约束，例如：

- 面板背景透明度
- 边框颜色和粗细
- 阴影/发光规则
- 标题字号级差

避免后续页面出现“新物种组件”。

## 几何系统必须锁定

高质量商业 Deck 的稳定性，很多时候不是输在颜色和字体，而是输在几何关系。

至少锁定以下几类几何规则：

1. 页面主区块边界
   - 这页真正的主体区域从哪里开始，到哪里结束
   - 主体区必须在视觉上居中，而不是“数学上差不多”

2. archetype 占比
   - 封面页：左文右图如何分区
   - 三卡摘要页：卡组总宽度和高度的合理范围
   - 时间轴页：纵轴高度、节点间距、文本卡片离轴距离
   - 结构页：主图占比不能过小，否则会显得空

3. 对齐轴
   - 同组卡片是否共边
   - 圆点与中轴线是否严格同心
   - 标签和内容块是否基于同一列宽系统

4. 关系型连线
   - 连线不允许凭感觉画
   - 必须从节点中心、锚点或骨架中间线推导出来
   - 连线断裂、穿帮、偏离节点中心，都应视为结构错误

5. 留白预算
   - 留白是结果，不是目标
   - 如果信息量不足以支撑大卡片，应该缩短卡片高度，而不是放任空洞

## 页面不应自由手绘

对于代码型 PPT，Build AI 不应直接自由决定每个元素的 `x/y/w/h`。

更稳的方式是：

1. 先选 archetype
2. 再套 page skeleton
3. 再用组件填充
4. 最后只允许在局部范围内微调

否则很容易出现：

- 主体区不居中
- 卡片高度过大导致空
- 节点与线不共轴
- 同一页里出现不一致的列宽和 gutter
