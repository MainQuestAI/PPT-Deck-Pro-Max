# Layout Geometry Rules

## 目标

让 Deck 的稳定性不再依赖“设计感觉”，而是依赖一组可复用的几何规则。

这份规则主要解决四类问题：

1. 连线断裂
2. 主区块不居中
3. 节点与轴线不共心
4. 卡片高度与内容密度不匹配

## 核心原则

### 1. 先锁骨架，再填内容

每一页先定义：

- 页面安全区
- 主体区边界
- 主视觉边界
- 辅助区边界
- 底部说明区边界

没有骨架，不允许直接摆组件。

### 2. 先锁关系，再画连线

所有连接线必须来自以下任一来源：

1. 节点中心点
2. 卡片锚点
3. 中轴线交点

禁止“目测差不多”的自由线段。

### 3. archetype 必须有占比规则

不同 archetype 的留白预算不同，不允许一刀切。

## archetype 几何规则

### A. 封面页

- 左文区建议占 `48% - 58%`
- 右视觉区建议占 `28% - 36%`
- 右侧结构图必须形成一个完整闭环或连续关系，不允许局部悬空
- 标签区与主视觉区必须形成垂直逻辑，不可各自漂浮

### B. 三卡摘要页

- 三卡总宽度应占页面主体宽度 `72% - 84%`
- 卡片组整体必须相对于页面安全区居中
- 单卡高度如果超过 `3.6` 英寸，必须补充第二层信息或缩短高度
- 三卡页底部标签带只能作为辅助，不可承担主体视觉

### C. 时间轴 / 泳道页

- 中轴线必须穿过圆点几何中心，允许误差极小
- 所有节点卡片与中轴线的水平距离必须一致
- 同一时间轴内节点间距必须统一
- 左右双时间轴页必须使用对称 gutter

### D. 结构页 / 架构页

- 主图占页面主体区至少 `55%`
- 如果结构页主要信息只有 3 个节点，不应用大面积空容器撑版
- 分层结构页的层带高度不应完全平均，应按内容密度微调

### E. 表格 / 对比页

- 表头带与数据区之间必须有明确分层
- 表格左列宽度必须优先服务识别，不应与数值列平均分配
- 右侧解释卡片高度应跟随内容，而不是强行等高

## 页面骨架输出建议

在出图前，应额外生成：

- `deck_page_skeletons.md`

每页至少记录：

1. 页码
2. archetype
3. 主体区边界
4. 主视觉边界
5. 对齐轴
6. 组件组关系
7. 预期占比

## 需要进入 QA 的几何指标

1. 主体区中心偏差
2. 组件组总占比
3. 连线是否锚定在节点中心
4. 同组卡片左边/右边/顶部是否齐
5. 内部空白率是否过高

## 典型失败模式

### 连线断

原因：
- 连线使用固定坐标
- 节点移动后未同步

处理：
- 线段从节点锚点推导
- 禁止手写孤立线坐标

### 三卡页显空

原因：
- 卡片高度来自统一模板，不看实际文案量

处理：
- 给 archetype 加密度阈值
- 文案少时缩短卡片高度或加二级证据

### 时间轴不稳

原因：
- 圆点和中轴线没有共享计算基准

处理：
- 圆点中心必须直接使用中轴 x
- 卡片起点必须由统一 gutter 推导
