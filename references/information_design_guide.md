# Information Design Guide

## 目标

让 AI 看到任何一段业务内容时，能自动判断"这里的数据关系是什么"，然后选择最有冲击力的视觉形式。

**核心原则：Deck 中的每一页都应该有一个"视觉主角"。如果一页只有文字面板，说明信息设计步骤没做到位。**

## 数据关系 → 视觉形式映射

### 1. 对比关系（A vs B / 旧 vs 新）

当内容在做两种方案、两个阶段、两类做法的对比时：

| 场景 | 推荐视觉 | 模板引用 |
|------|---------|---------|
| 两方维度对比（4+ 维度） | 双栏表格 + **雷达图** | 无内置模板，用 SVG 绘制 |
| 两方简单优劣 | 双栏表格 + **差异高亮** | comparison_bar.html |
| 前后效果对比 | 横向 **柱状图**（传统 vs AI） | comparison_bar.html |

**关键判断：如果对比维度 ≥ 3，优先用雷达图；如果只是 2 方简单对比，用表格 + 高亮足够。**

### 2. 程度 / 缺口（X 有多大 / X 缺多少）

当内容描述某个能力的不足、覆盖率的差距、完成度的欠缺时：

| 场景 | 推荐视觉 | 说明 |
|------|---------|------|
| 3 个并列维度各有缺口 | **仪表盘（gauge）× 3** | 如"三层断裂"各有不同程度 |
| 单一指标的缺口 | **大数字 + 进度条** | 如"覆盖率仅 25%" |
| 多指标的当前水平 | **多条进度条** | 如"五项能力评估" |

**关键判断：只要内容描述了"不足/缺口/断裂/差距"，就应该用可量化的视觉形式。文字描述"不足"不如一个 25% 的仪表盘有冲击力。**

### 3. 阶段 / 流程（先后顺序）

当内容描述一个过程、步骤、演进路径时：

| 场景 | 推荐视觉 | 模板引用 |
|------|---------|---------|
| 3-5 步线性流程 | **icon 流程链**（带箭头） | stage_band.html |
| 时间维度的演进 | **时间线**（带阶段标记） | stage_band.html |
| 并行多路径 | **泳道图** | 无内置模板 |

**关键判断：流程链中的每一步都应该有一个 icon，不能只是文字节点。icon 让流程链从"列表"变成"图示"。**

### 4. 闭环 / 循环（A→B→C→A）

当内容描述一个持续运转的系统、反馈循环时：

| 场景 | 推荐视觉 | 模板引用 |
|------|---------|---------|
| 4-6 节点闭环 | **环形图**（每节点有 icon + 颜色） | closed_loop.html |
| 双向循环（如监测-修复） | **∞ 无穷环** | 无内置模板 |

**关键判断：闭环图的每个节点必须有专属 icon 和颜色，否则节点之间没有辨识度。**

### 5. 分类 / 并列（N 个同级概念）

当内容列举多个并列的模块、功能、价值时：

| 场景 | 推荐视觉 | 说明 |
|------|---------|------|
| 3-6 个模块/功能 | **icon 卡片网格** | 每卡有 icon + 标题 + 描述 |
| 5 个业务价值 | **彩色条形图/柱状图** | 每个价值一种颜色 |
| 3 个推荐场景 | **三栏卡片**（有 icon 头部） | 每栏有图标 + WHY + HOW |

**关键判断：并列概念 > 3 个时，必须用 icon 区分。纯文字列表在 Deck 中是"文档感"的来源。**

### 6. 层级 / 优先级

当内容描述优先级排序、层次结构时：

| 场景 | 推荐视觉 | 模板引用 |
|------|---------|---------|
| 3-4 层优先级 | **堆叠层级条** | priority_stack.html |
| 5 层 Agent 架构 | **树状层级图** | 无内置模板 |

### 7. 趋势 / 变化

当内容描述随时间的变化、效果对比时：

| 场景 | 推荐视觉 | 说明 |
|------|---------|------|
| 两条曲线对比 | **折线图**（传统 vs AI） | 如"会员经营效率趋势" |
| 增长预期 | **面积图** | 如"市场规模增长" |

### 8. 单一大数字（冲击力）

当内容有一个核心统计数据需要被记住时：

| 场景 | 推荐视觉 |
|------|---------|
| "6.02 亿用户" | 超大字号（80px+）metric card |
| "7 大模型监测" | 大数字 + 简短说明 |

**关键判断：如果一页有一个关键数字，让它成为视觉主角。80px 的"6.02 亿"比 14px 文字中提到"6.02 亿"冲击力大 10 倍。**

## 禁止事项

1. **禁止一页只有文字面板而没有任何视觉元素**（除非是封面或 CTA 页）
2. **禁止图表没有数据**——哪怕是说明性数据，也比空图表好
3. **禁止 icon 只用 emoji**——应使用 SVG icon 或 Unicode 图形符号
4. **禁止所有卡片大小一样、颜色一样**——必须有视觉重量差异
