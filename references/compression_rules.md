# Compression Rules

## 目标

把长文档压成页面表达，而不是把文档缩短一点点。

## 核心规则

1. 一页只保留一个主结论
2. 能变图形的，不保留成长段文字
3. 删除内部讨论语气
4. 删除重复推导和连接词
5. 不保留“为了完整而完整”的说明

## 证据保真原则

可以删掉：

- 形容词
- 连接词
- 重复解释
- 冗长推导

不能删掉：

1. 具体数字
2. 关键客户或品牌名称
3. 关键系统节点名称
4. 案例锚点

如果证据太多：

1. 先拆页
2. 再变图
3. 最后调整层级

不要为了版面好看而丢掉证据。

## 演讲备注

每页可选附带演讲备注。hero pages 的演讲备注是强制的。

格式（在 `deck_clean_pages.md` 的每页区块内）：

```markdown
> 演讲备注: 这一页的核心话术是"传统研究已经跟不上业务节奏"，重点强调时间成本和决策延迟。
```

演讲备注的内容应该是：

1. 这一页的核心话术或开场白
2. 需要重点强调的数字或案例
3. 预期的听众反应和过渡到下一页的衔接话术

演讲备注不是页面文案的重复。它是"站在台上该怎么说"的指引。
