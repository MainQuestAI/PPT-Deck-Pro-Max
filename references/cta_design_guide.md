# CTA Design Guide

## 目标

CTA 页是整套 Deck 最后一页，决定了读者是否采取下一步动作。它不是总结页。

## 三个必备要素

### 1. 门槛控制

让下一步动作看起来"容易跨出"，而不是"又一个大项目"。

```
❌ "启动 GEO 运营体系建设"
   → 读者反应："又一个大项目，算了"

✅ "选一个品类，跑一轮 4 周试点"
   → 读者反应："一个品类？4 周？可以试试"
```

门槛控制技巧：
- 缩小范围（"一个品类"而非"全品牌"）
- 缩短时间（"4 周"而非"季度项目"）
- 明确交付物（"基线报告 + 改善对比 + 扩展建议"）

### 2. 具体交付物承诺

读者要知道"做完后我手里会拿到什么"。

```
❌ "验证 AI 升级效果"
   → 模糊，不知道最后拿到什么

✅ "4 周后你会拿到：基线诊断报告 + 改善数据对比 + 下一阶段扩展建议"
   → 具体，可预期
```

### 3. 紧迫感

为什么是"现在"而不是"以后再说"。

紧迫感来源：
- 窗口期（"AI 搜索用户规模每季度增长 X%，越晚做越落后"）
- 竞品动态（"头部品牌已经在建 GEO 体系"）
- 先发优势（"谁先跑通闭环，谁先拿到数据壁垒"）
- 机会成本（"每延迟一个月，X 个高价值问题可能被竞品占位"）

## CTA 页结构建议

```
标题：判断句（"选一个品类，跑一轮试点"）

左侧：
  - 试点入口描述（范围、时间、投入）
  - 具体交付物列表
  - 为什么是现在（紧迫感）

右侧：
  - 推荐的试点场景（2-3 个选项，每个一句话说明为什么适合）

底部 Insight bar：
  - 升华句（"试点的目的不是证明 AI 很聪明，而是验证一条真实经营链路能否更有效"）
```

## 禁忌

- ❌ 把 CTA 做成总结页（重复前面说过的要点）
- ❌ 只有一句"联系我们"没有具体路径
- ❌ 门槛太高（"启动全面数字化转型"）
- ❌ 没有紧迫感（"有需要随时联系"）
