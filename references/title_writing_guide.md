# Title Writing Guide

## 目标

Deck 的标题是每页视觉面积最大的文字元素。好标题决定了读者是否继续看这页。

## 核心规则

### 1. 判断句，不是描述句

```
❌ 描述句："传统 CDP/MA 的主要瓶颈"
   → 读者反应："哦，要讲瓶颈"（被动接收）

✅ 判断句："系统越建越重，不是因为功能不够，而是三层能力没有打通"
   → 读者反应："为什么？"（主动追问）
```

判断句包含"是/不是"、"应该/不应该"、"因为/所以"——它在做决断，不是在开话题。

### 2. 最多 2 行，30 字以内

- 标题超过 2 行 = 这页想说的太多，应该拆页
- 超过 30 字 = 需要精炼表达

如果一句话说不完，先问自己：这页真正的"一句话结论"是什么？

### 3. 读者 3 秒测试

把标题单独拿出来，不看正文和图表。3 秒内读者能否理解这页在说什么？

- 能 → 标题合格
- 不能 → 标题太抽象或太长

### 4. 标题不能有内部语言

以下表述说明编排语言泄露了：
- ❌ "这一页负责…"
- ❌ "回答客户的顾虑"
- ❌ "建立 proof"

标题的主语必须是"业务问题/用户/品牌/结果"，不是"这一页"。

## 标题类型参考

| 类型 | 适用 | 示例 |
|------|------|------|
| 因果判断 | 诊断页 | "系统越建越重，是因为三层能力没打通" |
| 对比结论 | 差异化页 | "不是更好的服务商，是不同物种" |
| 行动号召 | CTA 页 | "选一个品类，跑一轮试点" |
| 数据冲击 | 数据页 | "6.02 亿用户已经在问 AI" |
| 场景画面 | 场景页 | "AI 让系统更像金牌导购，而不是自动发券机器" |
